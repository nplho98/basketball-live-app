# 計畫書：bug 修復＋精彩標記／休息畫面輪播／遙控計分 v1.0

- 日期：2026-07-16
- 背景：7/19（週六）友誼賽要實戰直播，bug 優先修，新功能接續
- 版本規劃：bug 修復＝v0.12.2（versionCode 89）；三項新功能＝v0.13.0（versionCode 90）

## 第一波：bug 修復（v0.12.2）

### bug 1：收播後 YouTube 端卡「直播中」很久
- 現象：APP 按直播結束，YouTube 觀看頁長時間仍顯示直播中。
- 查證方向：v0.12.0 的 `endLiveBroadcast()`（liveBroadcasts.transition→complete）是否真的執行成功——broadcastId 是否還在、API 例外是否被吞掉、呼叫時機是否在停止推流前就被 UI 流程跳過。
- 修法：transition 失敗自動重試 1 次（隔 5 秒）；仍失敗跳明確 Toast（含「請到 YouTube 工作室手動結束」）並寫入 log；成功也 Toast 告知「YouTube 直播已結束」。

### bug 2：直播中 APP 當掉跳出
- 現象：直播中整個 APP 閃退，重開顯示未在直播；無當機記錄可查。
- 修法（本輪先建觀測）：
  1. `Thread.setDefaultUncaughtExceptionHandler` 全域當機攔截，把完整堆疊寫進 APP 外部檔案區 `crashlog/`（檔名帶時間戳），寫完再交回系統預設處理（維持正常閃退行為不吞錯）。
  2. 下次啟動偵測到新 crashlog → Toast 提示已記錄。
  3. 順手審查 v0.12.0 新增碼的可疑點：前景服務通知更新的執行緒、背景切換時 Surface 流程、重連 coroutine 的生命週期。
- 已知限制：這輪只能「裝好黑盒子」，真正根因要等下次當機的記錄。

### bug 3：+3 動畫連按兩次，公牛撞擊發生在螢幕外
- 現象：連按 +3，第二輪播放時公牛的水平位移沒歸位，衝刺又疊加位移，撞擊點跑出畫面。
- 修法：重播前把濾鏡圖層的水平位置重置回起點（對照樣品頁 `atk` class 移除歸位的做法）；播放序號機制確認舊場次繪製全部作廢。

### bug 4：測速顯示 0M／曲線前段空白／有時整條沒有
- 根因（已從程式碼確認）：512KB 一塊、傳完才有取樣點——
  - 慢網路第一塊耗時數秒 → 曲線前段空白
  - 第一塊超過 10 秒 → 唯一取樣點落在圖表時間軸外 → 整條看不到
  - 上傳失敗記 0 → 只有零星樣本時建議值算出 0M
- 修法：
  1. `CURVE_CHUNK_BYTES` 512KB → 128KB（取樣密度×4）
  2. 開測先傳 64KB 暖身塊（TLS 握手/TCP slow start 不計入量測）
  3. 取樣點時間戳夾在 0~10 秒內再進圖表
  4. 有效（非 0）樣本為 0 個 → 顯示「測速失敗，請檢查網路」，不給 0M 建議

## 第二波：三項新功能（v0.13.0）——依「UI 空殼先行」原則

### 功能 A：精彩時刻標記 ⭐（原待辦 3）
- 直播畫面加 ⭐ 鈕：按下記「now − 回推秒數」的直播時間點＋當下節數/比分。
- 設定頁加「標記回推秒數」（預設 10 秒，5/10/15/20 下拉）。
- 收播後跳「精彩清單」：每筆可 ±5 秒微調、可刪除；一鍵複製 YouTube 章節格式（`mm:ss 第N節 信義+3` 逐行）。
- 標記資料存 JSON（隨場次），收播不清除，供休息輪播與日後剪輯用。

### 功能 B：休息畫面＝精華輪播（原待辦 4）
- 直播畫面加「休息畫面」鈕：按下→觀眾端切成回放模式；再按→回現場。
- 進休息畫面時：取「進入時間點往回最近 10 個標記」，每球播「標記點起 8 秒」，**順序舊到新**（預設，Boss 未拍板項）。
- **播完一輪停在靜態圖**（隊徽＋比分，兼顧降溫，Boss 未拍板項）；完全沒有標記時直接顯示靜態圖。
- 回放素材來自 v0.10.0 本地錄影檔（休息輪播需開啟錄影備份，未開啟時按鈕跳提示）。
- 技術路線：MediaMetadataRetriever/MediaCodec 抽幀 → 縮到 480x270、12~15fps → 走公牛動畫同一套 ImageObjectFilterRender 疊全畫面。
  - ponytail：先求能動，幀率/解析度是已知天花板，實測不順再降規格或升級解碼管線。
- 聲音：回放期間維持現場收音（不做錄影原聲混音）。

### 功能 C：遙控計分（原待辦 5）
- 直播手機內建輕量 HTTP 服務（ServerSocket 手寫，不加依賴）：第二支手機連同一網路（場館 Wi-Fi 或直播機熱點）後用瀏覽器開控制頁。
- 控制頁（APP 內嵌 HTML，單檔）：主/客 +1+2+3、-1、犯規 +/−、換節；操作即時反映到直播計分板。
- 直播畫面顯示連線資訊（IP:埠＋簡易 QR）；控制指令量極小（每按幾十 byte），不影響推流。
- 安全：只綁內網介面＋4 位數配對碼（開播隨機產生顯示在直播機上）。
- 不做藍牙版：Android 當 BLE 週邊裝置相容性差、第二支手機還得裝 APP；HTTP 版第二支手機零安裝。
