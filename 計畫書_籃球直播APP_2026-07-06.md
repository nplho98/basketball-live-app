# 計畫書：籃球直播 APP（推 YouTube＋畫面計分板）

**日期**：2026-07-06
**狀態**：待執行
**執行方式**：由 Sonnet 5 子代理開發，Fable 5 規劃與驗收
**專案資料夾**：`D:\Claude Code\TEST\project\14 Baskeball YT\`

## 背景與目標

Boss 要在籃球比賽現場用一支 Android 手機完成轉播：鏡頭拍攝比賽、直播推上 YouTube，畫面上燒入兩隊即時計分板，現場直接在手機上操作比分。第一版驗收標準：**能完整、穩定地播完一場比賽**（登入 → 設定 → 開播 → 全場計分/變焦/調曝光 → 收播，全程不斷流）。

## 需求清單（已與 Boss 確認）

1. 手機鏡頭直播，推流到 YouTube（RTMP）
2. 用 YouTube（Google）帳號登入，APP 內自動建立直播、設定標題與隱私（公開/不公開/私人），一鍵開播——不用手動貼串流金鑰
3. 計分板**燒進直播畫面**（觀眾看得到）：兩隊隊名、分數、節數、犯規數；每隊分數旁 +1 / +2 / +3 / -1 四顆按鈕
4. 相機可定焦（鎖定對焦）
5. 音量鍵＝畫面拉近拉遠（變焦）
6. 螢幕上有按鍵直接調直播影像亮度（鏡頭曝光補償，觀眾看到的亮度會變）
7. 串流規格可選：解析度、fps（24/30/60）、碼率（1400K/2300K/3300K/4000Kbps）、使用位元變動率（VBR）
8. 操作體驗：橫屏直播為主、防誤觸、直播中螢幕常亮

## 技術架構（方案 A）

- **語言/平台**：Kotlin，Android 原生 APP
- **推流引擎**：RootEncoder（開源 RTMP 推流庫；內建變焦、曝光、VBR、畫面疊加濾鏡，斷線重連）
- **計分板疊加**：以疊加圖層（OpenGL filter）將計分板渲染進影像流
- **YouTube 串接**：Google Sign-In 登入 + YouTube Data API v3（liveBroadcasts / liveStreams）自動建播、設標題與隱私、取得推流位址
- **打包**：Gradle 產 APK 側載安裝（不上 Play 商店）

## 執行步驟

1. **專案初始化**：建 Android 專案骨架、`.gitignore` 防呆（排除 keystore / apk / aab / 本地雜檔）
2. **UI 空殼先行**（Boss 通用規則）：先做三個畫面的空殼給 Boss 過目——
   - 直播主畫面（橫屏）：相機預覽＋計分板＋計分按鈕＋開播/收播＋曝光調整鍵
   - 設定頁：解析度 / fps / 碼率 / VBR、直播標題、隱私設定
   - 登入頁：Google 帳號登入
3. **Google API 憑證申請**：開 Google Cloud 專案、啟用 YouTube Data API、建 OAuth 憑證（需要 Boss 用自己的 Google 帳號配合按幾下，我會一步步帶）
4. **推流核心**：接上 RootEncoder，先用固定串流金鑰驗證能穩定推上 YouTube（內部開發驗證用，完成後移除）
5. **YouTube API 串接**：登入 → 自動建立直播 → 設標題/隱私 → 一鍵開播/收播
6. **計分板燒入畫面**：隊名、分數、節數、犯規；+1/+2/+3/-1 按鈕操作即時反映到直播畫面
7. **相機控制**：音量鍵變焦、定焦鎖定、曝光加減鍵
8. **操作體驗**：橫屏鎖定、直播中螢幕常亮、計分按鈕防誤觸（例如按鈕加確認間隔或鎖定開關）
9. **打包實測**：產 APK 裝到 Boss 手機，實際開一場「不公開」直播完整測試後驗收

## 預期結果

Boss 帶一支手機到球場，打開 APP → 登入過的帳號直接一鍵開播 → 全場邊拍邊按計分 → 觀眾在 YouTube 看到有比分的球賽直播 → 比賽結束按收播。

## 不在本次範圍內

- iPhone / iOS 版
- 直播同時本地錄影存檔
- 比賽計時器 / 進攻時鐘
- 多平台推流（FB、Twitch 等）
- 第二支手機遠端計分

## 已知風險 / 注意事項

- **OAuth 測試模式 7 天限制**：Google 憑證若停在「測試」狀態，登入授權每 7 天會過期要重新登入；屆時可把應用發布為正式版（自用會顯示未驗證警告但可略過）解決
- **上行頻寬**：體育館行動網路上傳若不穩，高碼率會斷流；APP 會做斷線重連，但現場建議先測速選擇合適碼率
- **手機發熱**：長時間 60fps 高碼率直播手機會熱，可能降頻；實測時要驗證整場續航
- **音量鍵攔截**：直播中攔截音量鍵做變焦後，音量本身改由系統面板調整
- **本機開發環境**：需要 Android Studio / Android SDK，若本機尚未安裝，第一步會先補裝
- **不同手機編碼相容性**：以 Boss 實際手機為準做實測，不追求全機型相容
