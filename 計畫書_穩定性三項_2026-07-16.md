- 日期：2026-07-16
- 目標版本：v0.12.0（versionCode 87）
- 範圍：Boss 拍板的三項一次做完，做完重建 debug APK 實測

## 項目 1：前景服務＋勿擾提醒

**目的**：直播中切回主畫面／切其他 APP／久放，不被系統殺進程斷播；防來電搶麥克風。

**做法**
1. Manifest 新增權限：`FOREGROUND_SERVICE`、`FOREGROUND_SERVICE_CAMERA`、`FOREGROUND_SERVICE_MICROPHONE`、`POST_NOTIFICATIONS`（Android 13+ 要跑 runtime 請求）。
2. 新增 `LiveForegroundService`（`foregroundServiceType="camera|microphone"`）：
   - 開播成功（onConnectionSuccess）→ `startForegroundService` 掛常駐通知「● 直播進行中」。
   - 收播／推流真正停止 → `stopForeground` + `stopSelf`。
   - Android 11+ 規定：服務必須在 APP 還在前景時啟動，之後才允許背景用相機/麥克風——所以在開播流程內啟動，不能等切背景才啟動。
3. 背景推流銜接（已知坑）：切背景時 `OpenGlView.surfaceDestroyed` 會停預覽，但推流編碼不能跟著停；回前景用 `replaceView(OpenGlView)` 接回本機預覽。實測驗證「切出去 3 分鐘再回來」推流不中斷、預覽正常。
4. 勿擾提醒：開播確認框流程中檢查 `NotificationManager.currentInterruptionFilter`，未開勿擾就跳提醒對話框（附「前往設定」按鈕＋「本次直播不再提醒」）。只提醒不強制。

## 項目 2：AE/AF 同步鎖定（點擊測光＋曝光鎖）

**目的**：點暗處＝對焦＋以該處測光（畫面增亮）；長按鎖定＝焦點與亮度一起凍結，直播中鏡頭在亮暗間移動不失焦、不跳亮度。行為對齊 Boss 之前用的直播 APP。

**前置查證（2026-07-16 反編譯 RootEncoder 2.6.1 已證實）**
- 函式庫沒有 AE_LOCK／AE_REGIONS API；`disableAutoExposure()` 是 AE_MODE_OFF 陷阱（亮度亂跳）不可用。
- 繞法：反射取私有欄位 `builderInputSurface`（CaptureRequest.Builder），設定後用 synthetic `access$applyRequest(manager, builder)` 套用到 repeating request。

**做法**
1. 開相機後查 `getCameraCharacteristics()`：`CONTROL_AE_LOCK_AVAILABLE`、`CONTROL_MAX_REGIONS_AE`。不支援→退回現行「只鎖對焦」，Toast 告知。
2. 點擊對焦（handleFocusGesture）追加：呼叫 `tapToFocus` 後，反射讀 builder 的 `CONTROL_AF_REGIONS`，原值複製到 `CONTROL_AE_REGIONS`（借函式庫算好的感光元件座標，不自己做座標換算），套用→測光跟著點擊處。
3. 長按鎖定：現有等待收斂流程結尾追加 `CONTROL_AE_LOCK=true`；解除鎖定（右側欄按鈕）同步設回 false。
4. 反射全程 try-catch：失敗→退回只鎖對焦＋Toast，不能讓直播炸掉。版本釘 2.6.1，升級函式庫時此段要重新驗證（已記入記憶）。

## 項目 3：斷線重連強化

**目的**：體育館網路閃斷不整場作廢——重試撐更久、網路一恢復立刻接回、YouTube 端不自動關台。

**做法**
1. 重試策略：`RECONNECT_MAX_RETRIES` 10 次 → 大幅拉長（實質上限改為「直到 Boss 手動收播」；重試間隔維持 5 秒）。重連中畫面維持「● 重新連線中…」＋已重試次數。
2. 網路恢復即重連：`ConnectivityManager.registerDefaultNetworkCallback`，重連等待中收到 `onAvailable` → 取消等待立即重試（不乾等 5 秒週期）。
3. YouTube 收播改手動：`YouTubeLiveRepository` `enableAutoStop=true → false`；收播流程改為停止推流後呼叫 `liveBroadcasts.transition(complete)` 正式結束直播。transition 失敗→提示 Boss 到 YouTube 工作室手動結束（寫進錯誤訊息，不靜默）。
4. 與項目 1 交互：重連期間前景服務通知文字同步顯示「重新連線中」。
