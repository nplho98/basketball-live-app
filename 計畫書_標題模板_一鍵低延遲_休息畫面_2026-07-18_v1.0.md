## 功能一：直播標題雙模板

### 需求
設定頁的直播標題改成「兩個模板」機制：
1. 兩個標題模板，**內容都可以編輯**並保存。
2. 開播前從兩個模板中**選一個**當直播標題。
3. 選定後標題欄可**微調文字**再開播——微調只影響本次標題，**不回寫模板**。

### 設計
- `StreamPrefs` 新增三個鍵：`title_template_1`、`title_template_2`、`title_template_selected`（1 或 2）。
- 設定頁直播標題區改版：
  - 模板一／模板二各一個輸入欄（EditText，可直接改內容，離開即存）。
  - 每個模板旁一個單選鈕（RadioButton）選用哪個。
  - 下方保留「本次直播標題」欄：切換選中模板時把模板文字帶入此欄，之後 Boss 可在此欄微調；開播實際採用**此欄的最終文字**。
  - 手動改過「本次直播標題」後再切模板 → 直接以新選模板文字覆蓋此欄（以模板選擇為準，避免狀態糾結）。
- 相容既有流程：`YouTubeLiveRepository` 建立直播時取標題的入口不變，只是值來自上述最終標題欄。
- 預設值：模板一帶入現行既有的直播標題值（升級不破壞現況），模板二空字串。

## 功能二：「不延遲最低設定」一鍵套用

### 需求
設定頁**解析度項目上方**加一顆按鈕「不延遲最低設定」，按下直接套用：
- 解析度 **720P**
- 影格率 **30fps**
- 碼率 **2300Kbps**

### 設計
- `activity_settings.xml` 在解析度列上方插入一顆全寬按鈕，樣式沿用設定頁既有按鈕。
- 點擊：一次寫入三個既有偏好鍵（解析度/影格率/碼率），並同步刷新三個欄位的 UI 顯示，Toast 提示「已套用不延遲最低設定（720P／30fps／2300Kbps）」。
- 純套值，不動推流邏輯；下次開播自然生效（與手動逐項調整完全等價）。

## 功能三：休息畫面（黑底四節計分表，樣式09）

### 需求（Boss 已拍板）
1. 直播頁加一顆按鈕，**格式跟⭐標記鈕相同、位置在畫面左側對稱處**。
2. 按下 → **推給觀眾的直播畫面**切成黑底，只顯示四節計分表；**鏡頭暫停擷取降溫**。
3. 同一顆按鈕切換：休息中按鈕變「返回直播」，再按恢復鏡頭畫面。
4. 計分表＝Boss 選定的**樣式09「計分板同款完整面板」**：
   - 左欄：賽事名稱（設定頁現有兩欄，同計分板顯示邏輯）＋金色直分隔線。
   - 右側表格：橫列＝主／客隊（設定頁現有隊名），直欄＝第1～4節＋總分。
   - 每節分數；未打的節顯示「–」（半透明暖白）。
   - 總分＝**已打完節數的分數加總**，金底黑字方塊（同計分板比分方塊）。
   - 配色沿用計分板藏青金定案：藏青漸層底（#173868→#0D2140, α215）＋金細邊框（α56 255,214,140）＋頂緣金飾線（#F3CF7F/#E0A83F 漸層淡出）＋金分隔線（α90 243,207,127）＋暖白文字（246,239,224）＋金底黑字方塊（#F7DFA0→#CF9F45）。
   - 樣品參照：`樣品_休息畫面四節計分表_2026-07-18.html` 樣式09 區塊。

### 各節分數自動結算
- 資料：`quarterScores`——主客各 4 節的已結算分數（`StreamPrefs` 持久化，防閃退／重開恢復；開播確認框勾「重置計分板」時一併清空）。
- 結算時機＝**切節數當下**：
  - 節數 +1：把「目前總分 − 已結算各節加總」記為剛打完那一節的分數。
  - 節數 −1（誤切回退）：收回該節結算（清掉該節、分數回到未結算狀態），與 +1 對稱。
- 休息畫面上的總分＝已結算節分數加總＋尚未結算的當前節得分（＝就是計分板當下總分，兩隊各自）。
  - 註：休息通常發生在剛切完節之後，此時當前節得分為 0，總分即已打完節加總，符合 Boss 需求描述。

### 畫面切換與鏡頭降溫
- 新類別 `BreakScreenRenderer`（Canvas 繪 Bitmap，做法同計分板 overlay）：產生 1080p（依直播解析度）黑底＋樣式09 面板置中的整張畫面。
- 切休息：推流改送這張靜態畫面、**停止相機擷取**（降溫是本功能重點之一）；切回：恢復相機畫面。
- **查證點（開發時先驗證再全寫）**：RootEncoder 2.6.1 `RtmpCamera2`（Camera2Base 舊路線，非 StreamBase）串流中停相機的可行做法，候選順位：
  1. 框架原生支援串流中關相機、以 filter/靜態源續推（最理想）。
  2. 全螢幕蓋靜態計分表 filter ＋ 呼叫框架停相機 API（若停相機後 GL 仍持續出幀）。
  3. 退而求其次：相機不停，只全遮畫面（無降溫效果，需回報 Boss 知情）。
  - 注意既踩過的坑：OpenGlView.stop() 主執行緒 clear filterRenders 非執行緒安全；加/刪 filter 前確保無並發（見 memory 技術備忘）。
- 休息中維持：推流不中斷、前景服務、重連邏輯照常；⭐標記照常可按（回放秒數不受影響）。
- 按鈕只在**直播中**可用（未開播時隱藏或停用）。

### 按鈕 UI
- 位置：左下，與右下⭐標記鈕鏡像對稱（同排高、同尺寸 88dp 寬、同底圖樣式）。
- 文字：平時「休息畫面」；休息中「返回直播」（可加醒目底色區別，開發時比照⭐鈕樣式微調）。

## 不做的事（本輪範圍外）
- 不做加時（OT）節次——固定 4 節，總分欄照樣涵蓋當下總分。
- 不動 +3 特效／回放輪播等已終結路線。
- 不改錄影預設（過熱的錄影 4K 預設問題，Boss 尚未拍板，另案處理）。

## 驗收清單（Boss 實機）
1. 設定頁：兩個模板可編輯保存；切選模板帶入標題欄；微調標題開播後 YT 標題＝微調後文字；模板內容不被微調污染。
2. 設定頁：按「不延遲最低設定」→ 解析度/影格率/碼率三欄立即顯示 720P/30/2300。
3. 直播中：切節數後按「休息畫面」→ 觀眾端變黑底樣式09計分表、各節分數與總分正確；按「返回直播」→ 鏡頭畫面恢復、推流全程不中斷。
4. 休息期間手機溫度上升趨緩（相機有實際停止擷取才算數；若採退路方案需另行告知）。
5. 節數誤切回退再切回，各節分數不重複、不錯亂。
6. 強關 APP 重開，各節已結算分數仍在；開播勾「重置計分板」後各節分數歸零。
